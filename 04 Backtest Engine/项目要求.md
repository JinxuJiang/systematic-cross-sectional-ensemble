# 回测层项目要求文档

---

## 一、项目定位

**模块归属**: Module 4 - Backtester (回测与评估)

**核心职责**:
- 接收模型训练层的 `predictions.parquet` 输出
- 通过 Alphalens 进行因子有效性分析（IC/IR、分层收益）
- 通过 Backtrader 进行严格交易回测（考虑成本、滑点、停牌）
- 输出回测报告和绩效指标

**与上游模块的接口**:
```
03模型训练层/experiments/{exp_id}/predictions.parquet
            │
            ▼
    04回测层/
        ├── alphalens_eval.py    # 因子分析
        └── backtrader_eval.py   # 交易回测
```

## 二、输入数据说明

### 2.1 数据来源

| 文件路径 | 说明 | 来源 |
|---------|------|------|
| `03模型训练层/experiments/{exp_id}/predictions.parquet` | 预测结果 | 模型训练层输出 |
| `02因子库/processed_data/market_data/close.parquet` | 收盘价（用于计算净值） | 因子库 |

### 2.2 predictions.parquet 格式

```
┌────────────────┬─────────────┬─────────────┬────────────────┬─────────┐
│     date       │ stock_code  │ pred_score  │ actual_return  │ fold_id │
├────────────────┼─────────────┼─────────────┼────────────────┼─────────┤
│ 2013-07-02     │ 000001.SZ   │    0.85     │     0.03       │    0    │
│ 2013-07-02     │ 000002.SZ   │    0.72     │    -0.01       │    0    │
│ ...            │ ...         │    ...      │     ...        │   ...   │
└────────────────┴─────────────┴─────────────┴────────────────┴─────────┘
```

| 字段 | 类型 | 用途 |
|------|------|------|
| `date` | datetime | 截面日期 |
| `stock_code` | string | 股票代码 |
| `pred_score` | float | **核心**：模型预测的Alpha得分，用于排序选股 |
| `actual_return` | float | 实际未来20日收益率，用于验证和计算收益 |
| `fold_id` | int | Fold编号，可用于分析模型稳定性 |

## 三、Alphalens 因子分析

### 3.1 功能定位

**Alphalens 用于「因子有效性评估」**，回答以下问题：
- 预测分数（pred_score）和未来收益（actual_return）相关性如何？
- 按预测分数分组，各组收益是否单调？
- 因子的IC稳定性如何？

### 3.2 核心指标

| 指标 | 说明 | 评判标准 |
|------|------|----------|
| **IC (Information Coefficient)** | 预测分数与实际收益的相关系数 | IC > 0.03 有效，IC > 0.05 优秀 |
| **IR (Information Ratio)** | IC均值 / IC标准差 | IR > 0.5 稳定 |
| **分层收益** | 按pred_score分10组，Top组 vs Bottom组收益差 | 单调递增说明因子有效 |
| **换手率** | 相邻两期因子排序的变化 | 越低说明因子越稳定 |

### 3.3 数据准备

Alphalens 需要特定格式的输入：

```python
# factor: MultiIndex (date, asset) -> value
factor = predictions.set_index(['date', 'stock_code'])['pred_score']

# prices: DataFrame, index=date, columns=assets
prices = load_close_prices()  # 从close.parquet加载

# 调用Alphalens
factor_data = alphalens.utils.get_clean_factor_and_forward_returns(
    factor=factor,
    prices=prices,
    periods=(20,),  # 我们预测的是20日收益
    quantiles=10    # 分10组
)
```

### 3.4 输出报告

- **IC分析**：日IC序列、月均IC、IC分布
- **收益分析**：分层收益表、多空收益曲线
- ** turnover 分析**：因子稳定性

## 四、Backtrader 交易回测

### 4.1 功能定位

**Backtrader 用于「严格交易回测」**，模拟真实的投资过程：
- 每天根据 `pred_score` 选股调仓
- 考虑交易成本（手续费、滑点）
- 考虑市场摩擦（停牌、涨跌停无法成交）
- 输出净值曲线和风险指标

### 4.2 回测设定

| 项目 | 设定 |
|------|------|
| **调仓频率** | 每日（或按配置）|
| **选股数量** | 每天选 pred_score 最高的 N 只股票（如50只） |
| **权重分配** | 等权重 / 市值加权（待定） |
| **交易成本** | 单边 1.5‰（印花税+佣金+滑点） |
| **停牌处理** | 停牌股票无法买入/卖出 |
| **涨跌停** | 涨停无法买入，跌停无法卖出 |

### 4.4 输出指标

| 指标 | 说明 |
|------|------|
| **累计收益** | 回测期总收益率 |
| **年化收益** | 年化收益率 |
| **夏普比率** | 风险调整收益 |

## 五、目录结构（极简架构）

```
04回测层/
├── alphalens_eval.py      # Alphalens因子分析（单脚本搞定）
├── backtrader_eval.py     # Backtrader交易回测（单脚本搞定）
├── utils.py               # 公共工具：数据加载、路径处理
├── reports/               # 回测报告输出（运行时自动创建）
│   └── {exp_id}/
│       ├── alphalens_report.html
│       └── backtrader_report.html
└── 项目要求.md            # 本文档
```

**设计原则**：
- 极简：就3个Python文件，无复杂类继承
- 独立：alphalens_eval.py 和 backtrader_eval.py 可单独运行
- 复用：utils.py 统一处理数据加载

## 六、与上游模块的协作

### 6.1 工作流程

```
03模型训练层
    │
    ├── 完成训练 ──► predictions.parquet
    │
    ▼
04回测层
    │
    ├── Alphalens分析 ──► IC报告、分层收益图
    │
    └── Backtrader回测 ──► 净值曲线、风险指标
```

### 6.2 实验对应关系

模型训练层和回测层通过 `exp_id` 关联：

```bash
# 模型训练
python 03模型训练层/main_train.py --exp-id exp_001

# 回测分析（读取对应的predictions）
python 04回测层/alphalens_eval.py --exp-id exp_001
python 04回测层/backtrader_eval.py --exp-id exp_001
```
