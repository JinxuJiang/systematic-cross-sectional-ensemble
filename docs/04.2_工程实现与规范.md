## 三、开发注意事项（重点）

### 3.1 涨停过滤实现

**问题背景**：A股实行涨跌停限制，开盘涨停的股票无法买入。若回测不考虑此限制，会高估策略收益。

**实现细节**（backtrader.eval.py）：
```python
# 获取T+1日数据
future_dates = df_period[df_period.index > date].index.unique()
next_date = future_dates[0]
next_day_df = df_period.loc[next_date]

# 合并T日和T+1日数据
merged = current_slice[['stock_code', 'close']].merge(
    next_day_df[['stock_code', 'open']], 
    on='stock_code', 
    suffixes=('_t', '_t1')
)

# 计算开盘涨幅
merged['open_return'] = merged['open_t1'] / merged['close_t'] - 1

# 过滤开盘涨停（>=9.9%）
tradable_stocks = merged.loc[merged['open_return'] < 0.099, 'stock_code']
```

**关键约束**：
- 必须有T+1日数据才能过滤
- 9.9%阈值对应主板10%涨跌停（考虑精度误差）
- 科创板/创业板（20%涨跌停）需单独处理

### 3.2 止损逻辑实现

**设计**：每日开盘前检查持仓，触发止损则市价清仓。

```python
def next(self):
    # 每日检查止损（在调仓逻辑之前）
    for data in self.datas:
        pos = self.getposition(data)
        if pos.size > 0:
            cost_price = pos.price
            current_price = data.close[0]
            
            # 跌超15%止损
            if current_price < cost_price * (1 - 0.15):
                self.order_target_percent(data=data, target=0)
```

**为何放在调仓逻辑前**：止损是风险控制，优先级高于调仓信号。

### 3.3 仓位计算逻辑

**等权分配90%仓位**：
```python
valid_stocks = [s for s in buy_list if s in self.getdatanames()]
if valid_stocks:
    target_per = 0.90 / len(valid_stocks)
    for stock in valid_stocks:
        self.order_target_percent(data=data, target=target_per)
```

**为何用 order_target_percent**：
- 自动处理已有持仓的情况
- 自动计算需要买入/卖出的股数
- 考虑手续费后的实际可用资金

### 3.5 trades.csv 与 Backtrader 回测

- trades.csv是由策略逻辑运行之后生成的，可以用来查看什么时候买/卖，买/卖了什么、买/卖了多少
- Backtrader 回测是真实模拟每日每日运行的买卖，用来模拟回测
  - Backtrader 是t日出信号，t+1日开盘才真实操作

| 维度 | Backtrader引擎 | trades.csv分析 |
|:---|:---|:---|
| 计算基础 | 事件驱动模拟真实交易 | 直接使用模型输出的actual_return |
| 成交价格 | T+1开盘价（真实可执行） | 理论收益（假设完美执行） |
| 交易成本 | 含手续费(0.2%)、滑点(0.1%) | 无成本假设 |
| 仓位管理 | 等权分配、90%资金限制 | 等权分配、90%资金限制 |
| 涨停过滤 | 过滤开盘涨停股（无法买入） | 不过滤 |
| 止损机制 | 支持15%止损 | 不支持 |
| 计算速度 | 慢（逐日模拟） | 快（向量化） |
| 结果差异 | 偏低（成本扣除） | 偏高（理论最优） |

## 五、常见问题

### 5.1 Alphalens 报告中的 max_loss 警告

**现象**：`UserWarning: max_loss is set to 35%`

**原因**：部分股票在价格数据中缺失，导致因子数据被丢弃。

**处理**：当前设置 `max_loss=0.35`（35%容忍度），若超过此阈值会报错。

### 5.2 Backtrader 回测结果与预期不符

**排查步骤**：
1. 检查预测数据日期范围：`predictions['date'].min/max`
2. 检查回测区间设置：`STRATEGY_PARAMS['start_date']`
3. 检查调仓信号生成：`generate_signals` 打印的日志
4. 检查交易记录：`trades.csv` 中的实际成交

### 5.3 换手率过高

**原因**：
- 预测分数波动大 → 使用 `--use-smooth` 降低换手
- 调仓频率过高 → 调整 `rebalance-period`
- 选股数量过少 → 增加 `stocks_per_batch`

### 5.4 止损触发频繁

**排查**：
