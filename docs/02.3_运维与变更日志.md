## 一、关键检查点

### 1.1 因子分布检查

| 检查项 | 标准值 | 检查方法 | 不通过处理 |
|:---|:---|:---|:---|
| 均值 | -0.05 ~ 0.05 | `factor.mean()` | 检查清洗流程 |
| 标准差 | 0.95 ~ 1.05 | `factor.std()` | 重新标准化 |
| 最小值 | > -5 | `factor.min()` | 检查去极值 |
| 最大值 | < 5 | `factor.max()` | 检查去极值 |
| 偏度 | -1 ~ 1 | `factor.skew()` | 可接受 |
| 峰度 | < 10 | `factor.kurt()` | 可接受 |

### 1.2 缺失值检查

| 检查项 | 标准 | 检查方法 | 不通过处理 |
|:---|:---|:---|:---|
| 缺失值比例 | < 50% | `factor.isna().sum() / len(factor)` | 检查数据源 |
| 连续缺失天数 | < 30 | `factor.isna().groupby((~factor.isna()).cumsum()).sum()` | 检查停牌 |
| 全空股票数 | < 10% | `(factor.isna().sum(axis=0) == len(factor)).sum()` | 剔除全空 |

### 1.3 与下期收益相关性检查

| 检查项 | 标准 | 检查方法 | 说明 |
|:---|:---|:---|:---|
| IC均值 | abs(IC) > 0.02 | `factor.corr(return_next, method='spearman')` | 月度IC |
| IR比率 | abs(IR) > 0.1 | `IC_mean / IC_std` | 稳定性 |
| IC正占比 | > 45% | `(IC > 0).sum() / len(IC)` | 方向稳定 |
| 最大回撤 | < 0.5 | `IC.min()` | 极端情况 |

### 1.4 未来函数检查

| 检查项 | 方法 | 通过标准 |
|:---|:---|:---|
| ret20最后一天 | `ret20_df.iloc[-1].notna().sum()` | > 0（允许有值） |
| 清洗跨天泄露 | 检查pipeline是否每日独立 | 是 |
| PIT对齐 | 检查财报是否公告后使用 | 是 |

**关键区别**：
- ret1使用preClose计算，因此T日的ret1需要T+1的close确认，最后一天应为空
- ret20使用20天前的close，最后一天可以有值

### 1.5 中性化效果检查

| 检查项 | 标准 | 检查方法 |
|:---|:---|:---|
| 与行业相关性 | abs(corr) < 0.05 | `factor.groupby(industry).mean().std()` |
| 与市值相关性 | abs(corr) < 0.05 | `factor.corr(np.log(market_cap))` |
| 行业均值 | 各接近0 | `factor.groupby(industry).mean()` |

## 三、运维操作

### 3.1 新增因子流程

```
Step 1: 开发实现
     在对应家族文件添加 factor_xxx() 方法
     编写因果逻辑Docstring
     本地测试验证计算正确

Step 2: 验证检查
     运行未来函数检查（最后一天应为空/有值根据逻辑）
     运行分布检查（均值0，标准差1）
     运行IC检查（与下期收益相关性）

Step 3: 生产部署
     添加到 compute_all() 可用列表
     运行生产计算
     运行清洗流程
     更新因子清单文档
```

### 3.2 因子重算流程

```python
# 1. 删除旧数据
from pathlib import Path
factor_path = Path('processed_data/factors/technical/ret20.parquet')
factor_path.unlink(missing_ok=True)

# 2. 重新计算
from alpha_factory.technical.momentum import MomentumFactors
mf = MomentumFactors()
mf.factor_ret20(save=True)

# 3. 重新清洗
from processors.pipeline import clean_factor_wide
cleaned = clean_factor_wide(factor_df, industry_df, market_cap_df)
```

### 3.3 月度更新流程

**每月1日执行：**

```
1. 更新基础数据（01数据层）
    运行 monthly_update.py
   
2. 计算新增日期因子
    检测现有因子最新日期
    计算缺失日期
    增量计算（而非全量重算）
   
3. 清洗新数据
    只清洗新增日期
    拼接回旧数据
   
4. 验证
    检查日期连续性
    检查无未来函数
    检查分布正常
```

### 3.4 故障回滚

| 故障类型 | 回滚方案 |
|:---|:---|
| 因子计算错误 | 删除parquet文件，重新计算 |
| 清洗参数错误 | 重新运行clean_factor_wide |
| 数据污染 | 从01数据层重新生成 |
| 代码bug | 回退git版本，重新计算 |

### 3.5 完整计算流程（全量更新）

**方式一：分步执行（推荐，便于排查问题）**

```bash
# 1、准备行情、财务宽表数据
cd 02因子库/src/data_engine
python main_prepare_market_data.py
python main_prepare_financial_data.py

# 2、计算因子数据
cd 02因子库/src/alpha_factory/technical
python main_compute_technical.py

cd 02因子库/src/alpha_factory/financial
python main_compute_financial.py
```

**方式二：一键全量更新（覆盖已有数据）**

```bash
# 一个命令完成四个步骤：行情宽表 + 财务PIT对齐 + 技术因子 + 财务因子
python 02因子库/update_all.py
```

**两种方式对比：**

| 方式 | 适用场景 | 耗时 | 内存要求 |
|:---|:---|:---|:---|
| 分步执行 | 首次运行、排查问题 | ~30分钟 | 中等 |
| 一键更新 | 月度维护、快速重算 | ~30分钟 | **较高（16GB+）** |

**注意**：一键更新时建议关闭其他应用（B站、微信等），避免内存不足。
