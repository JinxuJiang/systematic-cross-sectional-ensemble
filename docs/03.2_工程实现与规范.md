```python
def _compute_splits(self):
    # 估算交易日数量（日历天 × 0.69）
    train_days = int(self.train_window.days * 0.69)
    valid_days = int(self.valid_window.days * 0.69)
    test_days = int(self.test_window.days * 0.69)
    step_days = int(self.step.days * 0.69)
    
    # 最小需要长度
    min_required = train_days + self.gap_train_valid + valid_days + self.gap_valid_test
    
    current_test_start_idx = start_idx + min_required
    
    while current_test_start_idx + test_days <= end_idx:
        # 计算测试集边界
        test_start_idx = current_test_start_idx
        test_end_idx = min(test_start_idx + test_days, end_idx)
        
        # 计算验证集边界（留出gap_valid_test）
        valid_end_idx = test_start_idx - self.gap_valid_test
        valid_start_idx = max(valid_end_idx - valid_days, start_idx)
        
        # 计算训练集边界（留出gap_train_valid）
        train_end_idx = valid_start_idx - self.gap_train_valid
        train_start_idx = max(train_end_idx - train_days, start_idx)
        
        splits.append((train_dates, valid_dates, test_dates))
        current_test_start_idx += step_days
```

### 1.4 多模型预测融合 (fuse_predictions.py)

**文件位置**: `03模型训练层/fuse_predictions.py`

**功能**: 动态IC加权融合多个模型的预测结果，支持任意数量模型（2个及以上）

**核心逻辑流**:

```
模型1 smoothed_predictions.parquet ──┐
模型2 smoothed_predictions.parquet ──┼──→ 股票交集 ──→ 排名标准化 ──→ IC加权融合 ──→ 融合预测
模型3 smoothed_predictions.parquet ──┘                      ↑
                                                            │
                                                   滞后IC权重计算
```

**融合算法步骤**:

| 步骤 | 函数 | 说明 |
|:---|:---|:---|
| 1 | `determine_split_date()` | 确定分界日期（最长horizon模型的test结束日期） |
| 2 | `merge_with_intersection()` | 每天取股票交集（所有模型都有的股票） |
| 3 | `rank_standardize()` | 每天截面排名标准化（pct_rank，转换到[0,1]区间） |
| 4 | `calc_daily_ic()` | 计算每日截面IC（Spearman秩相关） |
| 5 | `calc_lagged_weights()` | 计算滞后IC权重（lag=基准模型horizon） |
| 6 | `fuse_with_weights()` | Test期间：每天动态加权平均 |
| 7 | `fuse_with_fixed_weights()` | Live期间：固定使用Test最后一天权重 |

**滞后IC权重计算原理**:

```python
def calc_lagged_weights(daily_ic, lag):
    """
    第t天的权重 = 第0天到第t-lag天的IC累积均值（不包含最近lag天）
    
    原因: 避免使用未来信息，确保权重的可实现性
    
    示例 (lag=20):
    Day 0-19:  等权重 (1/n_models)
    Day 20:    使用Day 0的IC
    Day 21:    使用Day 0-1的IC均值
    Day 50:    使用Day 0-30的IC均值
    """
    if t < lag:
        w = [1.0 / n_models] * n_models  # 等权重
    else:
        hist_ic = daily_ic.iloc[:t-lag+1].mean()  # 历史IC均值
        hist_ic = hist_ic.clip(lower=0)            # 负IC截断为0
        w = hist_ic / hist_ic.sum()                # 归一化
```

**分界日期处理**:

```
时间轴:  2015    2020    2024    2025
         │       │       │       │
模型5d:  [=====Test=====][Live..]
模型20d: [=========Test=========][Live..]
         │               │
         └───────────────┘
           split_date (取20d模型的test结束)

融合Test:  <= split_date 的部分，使用动态IC权重
融合Live:  > split_date 的部分，固定使用test最后一天权重
```

**BaseModel接口定义**：

```python
class BaseModel(ABC):
    @abstractmethod
    def fit(self, X_train, y_train, X_valid=None, y_valid=None): pass
    
    @abstractmethod
    def predict(self, X_test): pass
    
    @abstractmethod
    def get_feature_importance(self): pass
    
    def save(self, path):  # 通用pickle保存
        with open(path, 'wb') as f:
            pickle.dump(self, f)
    
    @classmethod
    def load(cls, path):  # 通用pickle加载
        with open(path, 'rb') as f:
            return pickle.load(f)
```

```python
def _discretize_labels(self, y, n_bins=10):
    # 按日期分组，在每个截面内独立分桶
    def rank_within_group(group):
        return pd.qcut(group, q=n_bins, labels=False, duplicates='drop')
    
    y_discrete = y.groupby(level=0).transform(rank_within_group)
    return y_discrete  # 0~9的整数等级
```

| `smoothed_live_predictions.parquet` | 平滑后实盘预测 | ✅ EMA平滑 |

**EMA平滑原理**：

**1. 什么是半衰期（halflife）**

半衰期表示「权重衰减到一半所需的交易日数」。自适应计算基于预测分数的1阶自相关性：

```
公式: halflife = ln(0.5) / ln(autocorr)

其中:
- autocorr: 预测分数的1阶自相关系数（-1到1之间）
- ln(0.5) ≈ -0.693

示例:
- autocorr = 0.94 → halflife = ln(0.5)/ln(0.94) ≈ 11.2天
- autocorr = 0.97 → halflife = ln(0.5)/ln(0.97) ≈ 22.8天
```

**2. 自适应特性**

| 市场状态 | 自相关 | halflife | 平滑效果 |
|:---|:---|:---|:---|
| 趋势性强 | 高（0.95+） | 长（20+天） | 大窗口平滑，保留趋势 |
| 波动性大 | 低（0.90-） | 短（5-10天） | 小窗口平滑，快速响应 |

**3. EMA计算公式**

```
EMA_t = α × pred_t + (1-α) × EMA_{t-1}

其中:
- α = 1 - exp(-ln(2)/halflife)  （平滑系数）
- pred_t: 第t天的原始预测分数
- EMA_t: 第t天的平滑后分数
```

**4. 为什么用EMA平滑？**
- **减少噪声**：单日预测波动大，EMA过滤随机扰动
- **趋势保留**：近期数据权重高，远期数据权重指数衰减
- **自适应窗口**：根据数据自相关自动调整平滑强度
- **IC更稳定**：平滑后预测的IC通常波动更小、更可靠

## 三、开发注意事项（重点）

### 3.1 标签计算方式（V1修复核心）

**问题背景**：T日收盘后才能拿到完整数据计算因子，实盘只能在T+1日开盘买入

**两种计算方式对比**：

| 方式 | 买入时点 | 卖出时点 | 公式 | 实盘可执行性 |
|:---|:---|:---|:---|:---|
| 收盘价标签(旧版) | T日收盘 | T+20日收盘 | close[t+20]/close[t] - 1 | 差（无法T日收盘下单） |
| 开盘价标签(V1) | T+1开盘 | T+(horizon+1)开盘<br>（如20d为T+21） | open[t+(h+1)]/open[t+1] - 1 | ✅ 好（真实可执行） |

**代码实现**：

```python
def _compute_labels(self, dates):
    close_df = self._load_close_data()
    
    if self.use_open_price:
        open_df = self._load_open_data()
        
        for date in dates:
            idx = all_dates.get_loc(date)
            
            entry_idx = idx + 1  # T+1日
            exit_idx = entry_idx + self.label_horizon  # T+(horizon+1)日，如20d为T+21
            
            entry_price = open_df.loc[all_dates[entry_idx]]
            exit_price = open_df.loc[all_dates[exit_idx]]
            
            label = exit_price / entry_price - 1
    else:
        # 旧版逻辑...
```

**影响**：
- V1标签计算的收益会比旧版低（因为T+1开盘可能跳空）
- 但这更真实反映实盘可执行收益
- 回测结果更可信

### 3.2 双重Gap机制

**为什么需要Gap？**

场景：label_horizon=20，T+1买入，T+21（即T+(horizon+1)）卖出

```
无Gap（数据泄露）:
Train: ... 2024-01-31
Valid: 2024-02-01, 2024-02-02, ...

问题: Train最后一天(1/31)的标签需要用到2/21的开盘价
      但Valid从2/1开始，Train"看到"了Valid的未来！

有Gap=21（安全）:
Train: ... 2024-01-31
Gap:   2024-02-01 ~ 2024-02-29 (21个交易日)
Valid: 2024-03-01, 2024-03-02, ...

安全: Train标签用到2/21，Gap覆盖到2/29，Valid从3/1开始
```

**Gap大小计算**：

```python
default_gap = label_horizon + 1  # 20 + 1 = 21天

gap_train_valid = wf_config.get('gap_train_valid', default_gap)
gap_valid_test = wf_config.get('gap_valid_test', default_gap)
```

**切分算法中的Gap实现**：

```python
# 验证集边界（留出gap_valid_test个交易日）
valid_end_idx = test_start_idx - self.gap_valid_test

# 训练集边界（留出gap_train_valid个交易日）
train_end_idx = valid_start_idx - self.gap_train_valid
```

**泄露检查**：

```python
def verify_no_leakage(self):
    for fold_id, (train, valid, test) in self.get_splits():
        # 检查1: 训练集标签不泄露到验证集
        train_last_idx = self.dates.get_loc(train[-1])
        train_label_end_idx = train_last_idx + self.label_horizon + 1
        valid_first_idx = self.dates.get_loc(valid[0])
        
        if train_label_end_idx >= valid_first_idx:
            print(f"[FAIL] Fold {fold_id}: 训练集标签泄露到验证集！")
            all_passed = False
```

### 3.3 Walk-forward切分算法

**交易日 vs 日历日转换**：

```python
# 估算交易日数量（日历天 × 0.69）
# 假设一年252个交易日，365 × 0.69 ≈ 252
train_days = int(self.train_window.days * 0.69)  # 3Y ≈ 756天
valid_days = int(self.valid_window.days * 0.69)  # 6M ≈ 126天
test_days = int(self.test_window.days * 0.69)    # 3M ≈ 63天
step_days = int(self.step.days * 0.69)           # 3M ≈ 63天
```

**Fold数量估算**：

```
总交易日: ~3500天 (2010-2024)
最小需要: train + gap1 + valid + gap2 + test
        = 756 + 21 + 126 + 21 + 63 = 987天
可用长度: 3500 - 987 = 2513天
Fold数量: 2513 / 63 ≈ 40个
```

**滚动方式**：滑动窗口 vs 扩展窗口

| 方式 | 特点 | 本项目选择 |
|:---|:---|:---|
| 滑动窗口 | 训练集长度固定，窗口滑动 | ✅ 采用 |
| 扩展窗口 | 训练集不断增长，包含所有历史 | ❌ 未采用 |

理由：滑动窗口让模型更关注近期数据，适应市场变化

### 3.4 LambdaRank特殊处理

**标签离散化**：

```python
def _discretize_labels(self, y, n_bins=10):
    """
    将连续收益率转换为整数等级（0~9）
    在每个截面（日期）内独立分桶
    """
    def rank_within_group(group):
        try:
            # 按分位数分桶
            return pd.qcut(group, q=n_bins, labels=False, duplicates='drop')
        except ValueError:
            # 股票太少无法分桶，直接按排名映射
            ranks = group.rank(method='first')
            max_rank = ranks.max()
            return ((ranks - 1) / max_rank * n_bins).clip(0, n_bins - 1).astype(int)
    
    # 按日期分组应用
    return y.groupby(level=0).transform(rank_within_group)
```

**Group数据准备**：

```python
def _prepare_group_data(self, X):
    """
    LambdaRank需要知道每个query（交易日）有多少个document（股票）
    返回: groups = [3000, 3000, 2800, ...] 每个日期的股票数量
    """
    dates = X.index.get_level_values(0)
    date_counts = pd.Series(dates).value_counts().sort_index()
    groups = date_counts.values
    
    # 按日期排序X，确保groups和数据对齐
    X_sorted = X.sort_index(level=0)
    
    return X_sorted, groups
```

**训练数据构造**：

```python
# 标签离散化
y_train_discrete = self._discretize_labels(y_train, n_bins=10)

# 准备group数据
X_train_sorted, train_groups = self._prepare_group_data(X_train)
y_train_sorted = y_train_discrete.sort_index(level=0)

# 创建Dataset

    X_train_sorted.values,
    label=y_train_sorted.values,
    group=train_groups,
    feature_name=self.feature_names
)
```

### 3.6 多模型融合开发详解（fuse_predictions.py）

**融合核心逻辑**（7步流程）：

```
Step 1: 加载数据 → 读取各模型的smoothed_predictions.parquet
Step 2: 确定分界 → 取horizon最长模型的test结束日期作为split_date
Step 3: 股票交集 → 每天inner join取所有模型共同覆盖的股票
Step 4: 排名标准化 → 每天截面pct_rank，统一量纲到[0,1]
Step 5: 计算IC → 用基准模型的actual_return计算Spearman IC
Step 6: 滞后权重 → lag=基准horizon，前lag天等权，之后历史IC均值
Step 7: 加权融合 → Test期动态权重，Live期固定Test最后一天权重
```

---

#### 关键点1：用Base模型的Actual计算IC

**为什么要用base模型的actual？**

假设融合5d模型和20d模型：
- 5d模型actual = open[t+6]/open[t+1] - 1（持仓5天收益）
- 20d模型actual = open[t+21]/open[t+1] - 1（持仓20天收益）

**问题**：如果各自用自己的actual算IC
- 5d模型IC反映预测5天收益的能力
- 20d模型IC反映预测20天收益的能力
- 两者IC不可比（收益周期不同）

**解决方案**：
```
统一用base模型（如20d）的actual计算IC：
- 5d模型的IC = pred_5d vs actual_20d
- 20d模型的IC = pred_20d vs actual_20d

结果：两个IC都反映"预测20天收益"的能力，可直接比较
```

**实际意义**：
- base_idx参数决定调仓周期（通常选最长horizon）
- 融合权重反映各模型对「目标调仓周期」的预测能力

---

#### 关键点2：分界日期（split_date）处理

**为什么需要分界日期？**

不同horizon模型的test期长度不同：
```
60d模型: [=========Test=========]......
20d模型: [=============Test=============]
         ↑ split_date (60d test结束)
         │
         分界前：两模型都有actual_return → 可以算IC → 动态权重
         分界后：只有20d有actual_return → 不能算IC → 固定权重
```

**分界日期确定**：
```python
split_date = max(各模型test_end_date)  # 取最长horizon模型的test结束
```

**分界后的处理**：
- **Test_remain**（>split_date但短模型还有test）：
  - 长模型已只有live，短模型还有test
  - 使用Test最后一天权重（固定）
  - 这部分数据参与回测，但不参与IC权重计算
  
- **Live期**：
  - 所有模型都只有pred，没有actual
  - 固定使用Test最后一天的权重

**设计意义**：
- 最大化利用数据：分界后到长模型live开始前，短模型还能继续跑
- 避免未来信息：固定权重确保不"偷看"未来数据

---

#### 关键点3：滞后IC权重（lagged IC）

**为什么需要滞后？**

IC计算用到actual_return，而actual_return需要horizon天后的价格：
```
Day 20的IC计算：需要Day 20+horizon的价格
如果直接用Day 20的IC决定Day 20的权重 → 用到未来价格！

正确做法（lag=horizon）：
Day 20的权重 ← 使用Day 0的IC
Day 21的权重 ← 使用Day 0-1的IC均值
...
Day t的权重 ← 使用Day 0到Day t-horizon的IC均值
```

**权重计算规则**：
| 阶段 | 天数 | 权重计算方式 |
|:---|:---|:---|
| 初始化 | t < lag | 等权重 [1/n, 1/n, ...] |
| 累积期 | t ≥ lag | 历史IC均值（不含最近lag天）|
| 负IC处理 | - | clip(lower=0)，负IC截断为0 |

---

#### 重难点总结

| 问题 | 解决方案 | 原因 |
|:---|:---|:---|
| IC不可比 | 用base模型的actual统一计算 | 确保同一调仓周期比较 |
| Test长度不同 | 分界日期切分 | 长模型结束后短模型继续跑 |
| 权重泄露 | lag=horizon滞后 | IC计算用到未来价格 |
| Live无actual | 固定Test最后一天权重 | 无法计算新IC |
| 量纲不同 | pct_rank标准化 | 统一到[0,1]区间 |

## 六、常见问题

### 6.1 为什么IC比之前低？

**原因**：V1版本使用真实交易时点（开盘价）计算标签，而非收盘价

```
旧版: T日收盘买入 → T+20日收盘卖出（理想化，无法实盘执行）
V1:   T+1日开盘买入 → T+(horizon+1)日开盘卖出（真实可执行，如20d为T+21）

差异: T+1开盘可能跳空（高开/低开），导致收益降低
     但这是真实可执行的收益，回测更可信
```

**正常现象**：
- 旧版IC可能0.08-0.12
- V1版IC可能0.03-0.06
- 虽然数值降低，但实盘可信度大幅提升

### 6.2 Fold数量怎么确定？

**计算公式**：
```
Fold数量 = (总交易日数 - min_required) / step

min_required = train_days + gap_train_valid + valid_days + gap_valid_test + test_days
             = 756 + 21 + 126 + 21 + 63 = 987天

举例: 2010-2024共3500个交易日
Fold数量 = (3500 - 987) / 63 ≈ 40个
```

**影响因素**：
- start_date设置越早，Fold越多
- step越小，Fold越多
- train_window越小，Fold越多

### 6.4 如何防止过拟合？

**LightGBM过拟合常见表现**：
- 训练集loss持续下降，验证集loss上升
- IC波动大，某些Fold IC为负
- 特征重要性抖动严重

**防过拟合措施**：

| 措施 | 参数 | 建议值 |
|:---|:---|:---|
| 限制树深度 | max_depth | 5-7 |
| 增加叶子最小样本 | min_data_in_leaf | 50-150 |
| 降低学习率 | learning_rate | 0.01-0.03 |
| 列采样 | feature_fraction | 0.7-0.8 |
| 行采样 | bagging_fraction | 0.8 |
| L1正则 | lambda_l1 | 0.1-0.5 |
| L2正则 | lambda_l2 | 0.1-1.0 |
| 早停轮数 | early_stopping_rounds | 50 |

### 6.6 预测平滑、多模型融合相关问题

**问题1: 平滑（EMA）干嘛用？**

**目的**: 减少预测噪声，使信号更稳定

**原理**:
```
原始预测: 模型每天输出原始分数，受噪声影响大
EMA平滑: 对历史预测做指数加权平均，近期权重高，远期权重低

效果:
- 原始预测: [0.5, 0.3, 0.6, 0.2, 0.7] → 波动大
- EMA平滑: [0.5, 0.42, 0.51, 0.38, 0.52] → 更平滑
```

**halflife计算**:
- 基于预测分数的1阶自相关系数
- 自相关高 → halflife长 → 平滑窗口大（趋势性强）
- 自相关低 → halflife短 → 平滑窗口小（波动性强）

**为什么用平滑后预测做融合？**
- 平滑后预测IC通常更稳定
- 减少极端值对融合权重的干扰
- 实盘信号更可靠

---

**问题2: IC权重如何计算？**

**核心逻辑**: 用历史IC表现决定权重，但防止使用未来信息

**计算步骤**:

1. **每日IC计算**（Spearman秩相关）
   ```
   Day t IC = corr( pred_rank_t , actual_return_rank_t )
   ```

2. **滞后权重**（lag = base模型horizon）
   ```
   Day t 权重 ← 使用 Day 0 到 Day t-lag 的IC均值
   
   示例 (lag=20):
   Day 0-19:  等权重 [0.5, 0.5]（历史不足）
   Day 20:    用 Day 0 的IC
   Day 21:    用 Day 0-1 的IC均值
   Day 50:    用 Day 0-30 的IC均值
   ```

3. **负IC截断**
   ```
   历史IC < 0 → 置为0（负相关模型不给权重）
   所有IC ≤ 0 → 回退等权重
   ```

4. **归一化**
   ```
   权重_i = 截断后IC_i / sum(截断后IC)
   ```

**为什么需要lag？**
- IC计算用到actual_return，actual需要horizon天后才能确定
- 如果没有lag，Day 20的权重会用到Day 20+horizon的价格 → 未来信息泄露
- lag确保权重计算只用「已确定」的历史IC

---

**问题3: 融合权重为什么是动态的？**

**原因**: 不同模型在不同时期表现不同，动态权重可以自动适应市场变化

```
上涨市场: 长周期模型(20d/60d)可能表现更好 → 权重自动偏向长周期
下跌市场: 短周期模型(5d)可能更灵活 → 权重自动偏向短周期
震荡市场: 各模型分散配置 → 权重相对均衡
```

**问题4: lag参数如何选择？**

**原则**: lag应等于基准模型的horizon

| 基准模型 | horizon | 建议lag | 理由 |
|:---|:---|:---|:---|
| 5d模型 | 5 | 5 | 短周期快速响应 |
| 20d模型 | 20 | 20 | 平衡稳定性与响应速度 |
| 60d模型 | 60 | 60 | 长周期更稳定 |

**问题3: 融合后IC反而降低？**

**正常现象**:
- 单个模型可能在某些时期表现极好（过拟合）
- 融合后平滑了这种极端表现
- 但融合的稳健性更好，实盘更可靠

**检查方法**:
```python
# 比较各模型IC稳定性
for exp in ['test_001_hor5_v1', 'test_001_fined_20_v1', 'ensemble_5d_20d_v1']:
    df = pd.read_parquet(f'03模型训练层/experiments/{exp}/smoothed_predictions.parquet')
    daily_ic = df.groupby('date').apply(lambda x: x['pred_score_smooth'].corr(x['actual_return'], method='spearman'))
    print(f"{exp}: 平均IC={daily_ic.mean():.4f}, IC标准差={daily_ic.std():.4f}")
```

**问题5: 如何选择融合的base-idx？**

**建议**:
- 选择horizon适中的模型作为基准（通常是20d）
- 避免选择最短horizon（权重变化太快）
- 避免选择最长horizon（权重变化太慢）

```bash
# 推荐：以20d模型为基准
--exps test_001_hor5_v1 test_001_fined_20_v1 test_001_hor60_v1 --base-idx 1

# 不推荐：以5d模型为基准（权重变化太频繁）
--exps test_001_hor5_v1 test_001_fined_20_v1 test_001_hor60_v1 --base-idx 0
```
