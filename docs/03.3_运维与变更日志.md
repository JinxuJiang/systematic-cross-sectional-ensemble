## 一、关键检查点

### 1.1 模型训练检查

| 检查项 | 标准值 | 检查方法 | 不通过处理 |
|:---|:---|:---|:---|
| Fold数量 | > 10 | `splitter.get_n_splits()` | 调整start_date/step |
| 训练样本数 | > 10000/每Fold | `len(X_train)` | 检查数据完整性 |
| 验证样本数 | > 1000/每Fold | `len(X_valid)` | 减小valid_window |
| 测试样本数 | > 1000/每Fold | `len(X_test)` | 减小test_window |
| 训练完成 | 无异常 | 查看日志 | 检查内存/数据 |

### 1.2 IC/Rank IC稳定性检查

| 检查项 | 标准 | 检查方法 | 说明 |
|:---|:---|:---|:---|
| 平均IC | 0.02 ~ 0.10 | `summary_df['ic'].mean()` | V1版0.03-0.06为正常 |
| IC标准差 | < 0.05 | `summary_df['ic'].std()` | 波动过大说明不稳定 |
| IC>0占比 | > 60% | `(summary_df['ic'] > 0).mean()` | 方向一致性 |
| IC>0.05占比 | > 30% | `(summary_df['ic'] > 0.05).mean()` | 有效信号比例 |
| 平均Rank IC | 0.02 ~ 0.10 | `summary_df['rank_ic'].mean()` | Spearman相关 |
| Rank IC > 0占比 | > 60% | `(summary_df['rank_ic'] > 0).mean()` | 方向一致性 |

**IC解读**：

| IC范围 | 含义 | 行动建议 |
|:---|:---|:---|
| > 0.10 | 优秀 | 值得深入研究和实盘测试 |
| 0.05 ~ 0.10 | 良好 | 可接受，继续优化 |
| 0.02 ~ 0.05 | 弱信号 | 需要特征工程或调整模型 |
| < 0.02 | 无效 | 检查数据/标签/模型 |
| < 0 | 反向 | 考虑反向操作或检查bug |

### 1.3 特征重要性稳定性检查

| 检查项 | 标准 | 检查方法 | 不通过处理 |
|:---|:---|:---|:---|
| Top5特征一致性 | > 3个相同 | 对比相邻Fold Top5 | 降低学习率/增加正则化 |
| 重要性分布 | 无单一特征垄断 | 最大重要性 < 30% | 检查是否有过强因子 |
| 重要性非零 | 所有特征都有贡献 | `importance > 0` | 剔除无用特征 |

**查看Top特征**：

```python
import pandas as pd
import glob

# 加载所有Fold的特征重要性
importance_files = glob.glob('experiments/{exp_id}/feature_importance/*.csv')
all_importance = []

for f in importance_files:
    fold_id = int(f.split('fold_')[1].split('.')[0])
    df = pd.read_csv(f, index_col=0)
    df['fold_id'] = fold_id
    all_importance.append(df)

importance_df = pd.concat(all_importance)

# 计算平均重要性
mean_importance = importance_df.groupby(level=0)['importance'].mean().sort_values(ascending=False)
print("Top 10 特征（平均重要性）：")
print(mean_importance.head(10))

# 检查稳定性
print("\n各Fold Top 5特征：")
for fold_id in sorted(importance_df['fold_id'].unique()[:5]):
    fold_imp = importance_df[importance_df['fold_id'] == fold_id]
    top5 = fold_imp.sort_values('importance', ascending=False).head(5).index.tolist()
    print(f"Fold {fold_id}: {top5}")
```

### 1.4 预测值分布检查

| 检查项 | 标准 | 检查方法 | 不通过处理 |
|:---|:---|:---|:---|
| 预测值范围 | 无极端异常 | `pred_score.min/max` | 检查去极值 |
| 预测值标准差 | > 0.001 | `pred_score.std()` | 换LambdaRank/调参 |
| 唯一值比例 | > 50% | `nunique / len` | 模型输出过于集中 |
| 截面排名区分度 | 能区分10组 | alphalens分位数 | 调整模型参数 |

**预测值分布检查代码**：

```python
import pandas as pd

pred_df = pd.read_parquet('experiments/{exp_id}/predictions.parquet')

# 整体分布
print("预测值分布：")
print(pred_df['pred_score'].describe())

# 按日期分组统计
print("\n每日预测值标准差统计：")
daily_std = pred_df.groupby('date')['pred_score'].std()
print(daily_std.describe())

# 检查是否过于集中
unique_ratio = pred_df['pred_score'].nunique() / len(pred_df)
print(f"\n唯一值比例: {unique_ratio:.2%}")
if unique_ratio < 0.5:
    print("警告: 预测值过于集中，建议换用LambdaRank或调整参数")
```

### 1.5 数据泄露检查

| 检查项 | 方法 | 通过标准 |
|:---|:---|:---|
| Gap设置验证 | `splitter.verify_no_leakage()` | 无FAIL输出 |
| 训练-验证无重叠 | 检查日期范围 | train_end < valid_start - gap |
| 验证-测试无重叠 | 检查日期范围 | valid_end < test_start - gap |
| 标签不跨集 | 检查label计算范围 | train_label_end < valid_start |

**运行泄露检查**：

```python
from dataset.splitter_v1 import WalkForwardSplitterV1

splitter = WalkForwardSplitterV1(...)
splitter.verify_no_leakage()  # 应输出 [PASS]
```

## 二、性能基准

### 2.1 训练耗时基准（2026-03-24，i7-12700H）

| 配置 | 数据量 | Fold数 | 单Fold耗时 | 总耗时 | 内存峰值 |
|:---|:---|:---|:---|:---|:---|
| default_config | 5000只×3500日 | ~40 | 30-60秒 | 20-40分钟 | 4-6GB |
| rank_config | 同上 | ~40 | 45-90秒 | 30-60分钟 | 4-6GB |
| fined_lgbm_config | 同上 | ~40 | 60-120秒 | 40-80分钟 | 4-6GB |
| horizon5_config | 同上 | ~40 | 30-60秒 | 20-40分钟 | 4-6GB |

**影响因素**：
- 因子数量越多，耗时越长
- n_estimators越大，耗时越长
- LambdaRank比回归慢（约1.5倍）

### 2.2 存储空间占用

| 数据类型 | 文件数 | 总大小 | 说明 |
|:---|:---|:---|:---|
| 单个实验 | 1个目录 | 500MB-2GB | 含所有Fold模型 |
| 单个模型 | 1个.pkl | 10-50MB | 取决于特征数量 |
| predictions.parquet | 1个 | 50-200MB | 所有Fold预测结果 |
| summary.parquet | 1个 | < 1MB | 汇总指标 |
| 特征重要性 | N个.csv | 每Fold < 100KB | 每个Fold一个文件 |
| 可视化图表 | 2个.png | 每图 < 500KB | IC趋势图、重要性图 |

### 2.3 内存占用

| 阶段 | 峰值内存 | 说明 |
|:---|:---|:---|
| 初始化DataConstructor | ~500MB | 加载配置，扫描因子文件 |
| 构造单个Fold数据 | ~2GB | 加载训练/验证/测试数据 |
| 训练模型 | ~3GB | LightGBM内部内存 |
| 同时构造多个Fold | ~4-6GB | 垃圾回收不及时时 |

**内存优化建议**：
- 关闭其他应用（浏览器、微信等）
- 定期重启Python内核
- 减小train_window（减少历史数据）

## 三、运维操作

### 3.1 新增实验流程

```
Step 1: 准备配置
     复制现有config，如 cp configs/default_config.yaml configs/my_config.yaml
     修改关键参数（learning_rate, max_depth等）
     
Step 2: 测试验证
     运行小规模测试（缩短start_date，减少Fold数）
     python main_train_v1.py --config configs/my_config.yaml --exp-id test_small --start-date 2022-01-01 -y
     
Step 3: 检查指标
     查看summary.parquet，确认IC正常
     查看特征重要性，确认无异常
     
Step 4: 全量训练
     修改start_date为2010-01-01
     python main_train_v1.py --config configs/my_config.yaml --exp-id my_exp -y
     
Step 5: 保存结果
     记录实验ID和关键指标
     对比不同配置的效果
```

### 3.2 复现实验流程

```python
# 从config.yaml复现实验
import yaml
from training.walk_forward_trainer_v1 import WalkForwardTrainerV1

# 加载已保存的配置
with open('experiments/{exp_id}/config.yaml', 'r') as f:
    config = yaml.safe_load(f)

# 创建训练器（会覆盖原实验目录，请备份）
trainer = WalkForwardTrainerV1(config, exp_id='{exp_id}_reproduce')
trainer.run()
```

### 3.3 模型对比流程

```python
import pandas as pd
import glob

# 收集多个实验的summary
exp_ids = ['exp_001', 'exp_002', 'exp_003']
comparison = []

for exp_id in exp_ids:
    summary = pd.read_parquet(f'experiments/{exp_id}/summary.parquet')
    comparison.append({
        'exp_id': exp_id,
        'avg_ic': summary['ic'].mean(),
        'std_ic': summary['ic'].std(),
        'avg_rank_ic': summary['rank_ic'].mean(),
        'ic_positive_ratio': (summary['ic'] > 0).mean(),
        'n_folds': len(summary)
    })

comparison_df = pd.DataFrame(comparison)
print(comparison_df.to_string(index=False))
```

### 3.4 故障回滚

| 故障类型 | 回滚方案 |
|:---|:---|
| 实验配置错误 | 删除experiments/{exp_id}/目录，重新运行 |
| 数据不匹配 | 检查02层因子数据，重新运行main_train_v1.py |
| 模型损坏 | 从其他Fold复制模型，或使用上一个Fold的模型 |
| 内存溢出 | 减小train_window，或分批训练 |
| 训练中断 | 删除不完整目录，重新运行（会自动生成新exp_id）|

**删除实验目录**：

```bash
# 删除指定实验
rm -rf 03模型训练层/experiments/{exp_id}/

# 批量删除旧实验（保留最近5个）
cd 03模型训练层/experiments/
ls -t | tail -n +6 | xargs rm -rf
```
