## 一、关键检查点

### 1.1 Alphalens 分析检查项

| 检查项 | 检查方法 | 通过标准 | 失败处理 |
|:---|:---|:---|:---|
| Rank IC 均值 | 查看报告或日志 | > 0.03 | 检查模型预测质量 |
| IR (信息比率) | 查看报告或日志 | > 0.5 | 因子不稳定，考虑平滑处理 |
| 分层收益单调性 | 查看returns_analysis.png | Top组 > Bottom组 | 检查预测逻辑 |
| 换手率 | 查看turnover_analysis.png | 日均换手 < 50% | 使用平滑预测降低换手 |
| 数据覆盖率 | 检查max_loss警告 | < 35% | 补充缺失的行情数据 |

**快速检查脚本**：
```bash
# 运行分析并检查关键指标
python alphalens_analysis.py --exp-id exp_001 2>&1 | grep -E "(Rank IC|IR:|Long-Short)"
```

### 1.2 Backtrader 回测检查项

| 检查项 | 检查方法 | 通过标准 | 失败处理 |
|:---|:---|:---|:---|
| 夏普比率 | performance.json | > 1.0 | 优化模型或调整参数 |
| 最大回撤 | performance.json | < 20% | 加强止损或降低仓位 |
| 胜率 | performance.json | > 45% | 检查选股逻辑 |
| 总交易次数 | performance.json | > 100 | 检查调仓频率设置 |
| 涨停过滤率 | 查看日志输出 | < 20% | 正常范围 |
| 止损触发次数 | 查看日志输出 | 适度 | 过多则检查选股质量 |

### 1.3 数据质量检查

```python
# 检查预测数据完整性
from utils import load_predictions

df = load_predictions('exp_001')
```

## 二、性能基准

### 2.1 Alphalens 分析性能

| 数据规模 | 耗时 | 内存占用 |
|:---|:---|:---|
| 1000只股票 × 1年 | ~30秒 | ~500MB |
| 3000只股票 × 5年 | ~2分钟 | ~2GB |
| 5000只股票 × 10年 | ~5分钟 | ~4GB |

**优化建议**：
- 减少quantiles数量（默认10，可改为5）
- 减少periods数量（默认(20,)，可只保留一个周期）

### 2.2 Backtrader 回测性能

| 数据规模 | 耗时 | 说明 |
|:---|:---|:---|
| 20只股票 × 1年 | ~10秒 | 仅涉及实际交易的股票 |
| 50只股票 × 5年 | ~1分钟 | 包含信号生成+回测 |
| 100只股票 × 10年 | ~3分钟 | 完整回测 |

**性能瓶颈**：
- 数据加载：首次加载parquet文件较慢
- 回测引擎：股票数量增加时线性增长
- 报告生成：matplotlib绘图耗时

## 三、运维操作步骤

### 3.1 新增回测实验

