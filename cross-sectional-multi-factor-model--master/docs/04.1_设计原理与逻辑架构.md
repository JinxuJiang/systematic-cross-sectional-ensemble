# 04.1 设计原理与逻辑架构
> 回测层 - 预测分数验证与策略绩效评估

---

## 一、层级职责

回测层是截面多因子模型的**最终验证环节**，核心职责包括：

| 职责 | 说明 |
|:---|:---|
| 因子有效性检验 | 使用 Alphalens 验证预测分数与未来收益的相关性（IC/IR分析） |
| 策略回测 | 使用 Backtrader 模拟实盘交易，计算净值曲线和风险指标 |
| 信号生成 | 将模型预测转化为可执行的交易指令 |
| 风险控制 | 组合优化、止损、仓位管理 |

**核心价值**：回答"模型预测能否转化为实际收益"这个问题。

---

## 二、架构设计

回测层采用**双轨验证架构**：

```
模型预测分数 (predictions.parquet)
           │
           ├──→ Alphalens 分析层 ──→ 因子层面验证
           │                              │
           │                         • Rank IC / IR
           │                         • 分层收益（单调性）
           │                         • 换手率/稳定性
           │                              │
           │                              ▼
           │                    ┌──────────────────┐
           │                    │ 因子是否有效？    │
           │                    │ IC > 0.03？      │
           │                    │ IR > 0.5？       │
           │                    └────────┬─────────┘
           │                             │
           ▼                             ▼
    Backtrader 回测层 ──→ 策略层面验证
                              │
                         • 调仓执行
                         • 成本/滑点
                         • 止损风控
                              │
                              ▼
                    ┌──────────────────┐
                    │ 策略是否可盈利？  │
                    │ 夏普 > 1？       │
                    │ 回撤 < 20%？     │
                    └──────────────────┘
```

### 2.1 核心模块

| 模块 | 文件 | 职责 |
|:---|:---|:---|
| Alphalens 分析 | alphalens_analysis.py | 因子有效性检验、IC/IR计算、分层收益分析 |
| Backtrader 回测 | backtrader.eval.py | 策略回测引擎、交易模拟、绩效计算 |
| 工具函数 | utils.py | 数据加载、路径处理、格式转换 |

## 六、核心设计决策

### 6.1 为何使用 Alphalens + Backtrader 双验证？

| 维度 | Alphalens | Backtrader |
|:---|:---|:---|
| 关注点 | 因子预测能力 | 策略可执行性 |
| 核心指标 | IC/IR、分层收益 | 夏普比率、最大回撤 |
| 假设 | 理想化交易（无成本） | 真实交易（成本+滑点） |
| 用途 | 模型迭代参考 | 实盘前最终验证 |

**结论**：两者互补，缺一不可。IC高但实盘差 = 执行问题；IC低但实盘好 = 运气/过拟合。

### 6.2 调仓频率：每月第一个交易日

**设计理由**：
- 与模型预测周期匹配（预测未来20日收益）
- 降低换手率，减少交易成本
- 避免"幸运日"问题（固定调仓日更稳健）

### 6.3 涨停过滤逻辑

**问题**：回测时若选到涨停股，实际无法买入，导致回测收益虚高。

**解决方案**：
```python
# 计算T+1开盘涨幅
open_return = open_t1 / close_t - 1

# 过滤开盘涨停（>=9.9%）
if open_return < 0.099:  # 可买入
    include_in_portfolio()
```

### 6.4 平滑预测支持

**平滑逻辑**：半衰期加权（exponential decay）
```
pred_smooth(t) = α × pred(t) + (1-α) × pred_smooth(t-1)
```

**效果**：降低换手率，提升信号稳定性，IC可能略降但实盘更优。

## 七、关键概念解释

### 7.1 Rank IC (Spearman)

**定义**：预测分数排名与未来收益排名的Spearman相关系数。

| IC范围 | 评价 |
|:---|:---|
| > 0.05 | 优秀 |
| 0.03 ~ 0.05 | 良好 |
| < 0.03 | 一般 |

### 7.2 IR (Information Ratio)

**定义**：IC均值 / IC标准差，衡量因子稳定性。

| IR范围 | 评价 |
|:---|:---|
| > 0.7 | 优秀 |
| 0.5 ~ 0.7 | 良好 |
| < 0.5 | 一般 |

### 7.3 分层收益 (Quantile Return)

**方法**：每日按预测分数分10组，计算各组未来收益。

**判断标准**：
- Top组收益 >> Bottom组收益：单调性良好
- 各组收益呈单调递减：因子有效

### 7.4 因子换手率

**定义**：相邻两期因子排名的变化比例。

```
Turnover = 1 - Rank_Autocorrelation
```

**平滑预测的作用**：提高Rank Autocorrelation，降低Turnover。
