## 一、核心 API 与逻辑流

整体数据流与核心 API 的对应关系：

`
原始数据层(01数据)
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│  数据引擎层 (Data Engine)                                    │
│  核心类: MarketDataLoader / FinancialDataLoader / PITAligner │
│  职责: 长格式→宽表、财务TTM计算、PIT公告日对齐                │
│  输出: processed_data/market_data/*.parquet                  │
│        processed_data/financial_data/*.parquet               │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│  因子计算层 (Alpha Factory)                                  │
│  核心类: MomentumFactors / VolatilityFactors / ValuationFactors... │
│  职责: 从宽表计算原始因子值                                   │
│  方法: factor_xxx() → 返回因子矩阵                           │
│  暂存: processed_data/factors/technical/*.parquet            │
│        processed_data/factors/financial/*.parquet            │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│  清洗处理层 (Processors)                                     │

### 1.3 Pipeline 清洗流程

| 类/函数 | 文件路径 | 在数据流中的位置 | 职责 |
|:---|:---|:---|:---|
| clean_factor() | processors/pipeline.py | 数据流第3步 | 单截面一键清洗入口 |
| clean_factor_wide() | processors/pipeline.py | 数据流第3步 | 宽表批量清洗入口 |
| mad_winsorize() | processors/outlier.py | Pipeline第1步 | MAD去极值 |
| fill_missing() | processors/missing_value.py | Pipeline第2步 | 缺失值填补 |
| neutralize() | processors/neutralizer.py | Pipeline第3步 | OLS中性化 |
| zscore_standardize() | processors/standardizer.py | Pipeline第4步 | Z-Score标准化 |

**clean_factor vs clean_factor_wide 对比**：

| 函数 | 处理范围 | 输入 | 内部调用 |
|:---|:---|:---|:---|
| clean_factor | 1天（1个截面） | 3个 Series | 直接清洗（4步流程） |
| clean_factor_wide | N天（整个时间序列） | 3个 DataFrame | 循环调用 clean_factor |

**clean_factor_wide 的简化源码**：
`python
def clean_factor_wide(factor_df, industry_df, market_cap_df):
    result = pd.DataFrame(...)  # 准备空筐
    
    for date in factor_df.index:  # 遍历每一天
        # 取出当天的1根黄瓜
        day_factor = factor_df.loc[date]      # pd.Series（1天）
        day_industry = industry_df.loc[date]  # pd.Series（1天）
        day_cap = market_cap_df.loc[date]     # pd.Series（1天）
        
        # 调用 clean_factor 切这根黄瓜
        cleaned = clean_factor(day_factor, day_industry, day_cap)
        
        # 把切好的放回筐里
        result.loc[date] = cleaned
    
    return result  # 返回整筐切好的黄瓜
`

**Pipeline 调用链**：
`python
# 一键清洗（单截面）
clean_factor(factor, industry, market_cap, 
             steps=['outlier', 'missing', 'neutralize', 'standardize'])

# 内部执行顺序：
# 1. mad_winsorize(factor) → 去极值
# 2. fill_missing(result, industry, market_cap) → 填补缺失
# 3. neutralize(result, industry_clean, market_cap_clean) → 中性化
# 4. zscore_standardize(result) → 标准化
`

| 因子名 | 类型 | 计算公式 | 方向 |

| ret60 | 动量 | close[t]/close[t-60] - 1 | 动量 |
| ret120 | 动量 | close[t]/close[t-120] - 1 | 动量 |
| ret20_60 | 动量 | ret20 - ret60 | 加速度 |
| std20 | 波动率 | std(ret1, 20) | 反向 |
| std60 | 波动率 | std(ret1, 60) | 反向 |

## 三、开发注意事项（重点）

### 3.1 TTM 计算逻辑

**问题背景**：利润表、现金表都是流量表，数据是累计值(1季报=Q1，半年报=Q1+Q2)，需要转换为单季度再滚动4季度求和。

**正确算法**：
`
步骤1: 累计值 → 单季度值
  - 跨年第一季度(3月): 单季度 = 累计值(新年Q1)
  - 同年内其他季度: 单季度 = 当前累计 - 上期累计

步骤2: 单季度 → TTM
  - TTM(t) = sum(单季度[t-3:t+1])  # 最近4个季度
`

**代码实现**（financial_data_loader.py）：
`python
def _calculate_ttm_from_cumulative(self, records, value_field):
    # 1. 按报告期排序
    sorted_records = sorted(records, key=lambda x: x['report_date'])
    
    for i, record in enumerate(sorted_records):
        current_value = record[value_field]
        current_month = int(str(record['report_date'])[4:6])
        
        # 2. 计算单季度
        is_q1 = (current_month == 3)
        if is_q1 and 跨年:
            quarter_value = current_value  # 新年Q1，累计值重新计数
        else:
            quarter_value = current_value - prev_value  # 同年内相减
        
        # 3. 计算TTM（最近4个单季度之和）
        if i >= 3:
            ttm = sum(quarter_values[i-3:i+1])
`

**易错点**：
- 错误：直接用累计值/4 或相邻报告期简单相减
- 正确：必须判断跨年的Q1，此时累计值重新开始

### 3.2 PIT 对齐实现

**问题背景**：财务数据是季度公告，需要按公告日(m_anntime)对齐到交易日，避免未来函数。

**对齐逻辑**：
`
财报1(2010Q1, 4/29公告) ────── 财报2(2010Q2, 8/25公告)
       │                              │
       ▼                              ▼
  2010-04-29 ~ 2010-08-24       2010-08-25 ~ 2010-10-27
      使用财报1数据                使用财报2数据
`

**代码实现**（pit_aligner.py）：
`python
def align(self, records, date_field, value_fields):
    # 1. 按公告日排序
    valid_records.sort(key=lambda x: x[0])  # 按公告日排序
    
    # 2. 遍历交易日历，找到当前适用的最新财报
    for trade_date in self.trading_calendar:
        while (record_idx + 1 < len(valid_records) and 
               trade_date >= valid_records[record_idx + 1][0]):  # 公告日已到达
            record_idx += 1
            current_record = valid_records[record_idx]
        
        # 使用当前记录的财务数据
        result.append((trade_date,) + current_values)
`

**关键约束**：
- 数据层已去重（同一报告期保留最早公告日）
- PIT层只做对齐，不做清洗
- 对齐结果长度 = 交易日历长度

### 3.3 去极值参数选择

**MAD (Median Absolute Deviation) 算法**：
```python
median = factor.median()
mad = (factor - median).abs().median()
lower_bound = median - n_mad * constant * mad
upper_bound = median + n_mad * constant * mad
```

**参数选择**：

| 参数 | 默认值 | 说明 |
|:---|:---|:---|
| n_mad | 3.0 | MAD倍数，3倍约对应正态分布的3σ，覆盖99.7%数据 |
| constant | 1.4826 | 调整常数，使MAD与标准差在正态分布下一致 |

**为何选MAD而非3σ**：
- 中位数和MAD对极端值不敏感（稳健统计）
- 3σ受极端值影响大，可能缩尾不充分

### 3.4 OLS 中性化细节

**回归模型**：
```
因子 = β₀ + β₁×行业dummy₁ + ... + βₙ×行业dummyₙ + βₘ×对数市值 + ε
输出: ε (残差，即剥离行业+市值影响后的纯Alpha)
```

**实现细节**：

1. **对数市值计算**：
   ```python
   log_market_cap = np.log(market_cap.replace(0, np.nan))
   log_market_cap = log_market_cap.fillna(log_market_cap.median())
   ```

2. **小行业合并**：
   ```python
   min_industry_stocks = 2  # 最少2只股票
   small_industries = industry_counts[industry_counts < min_industry_stocks].index
   industry_valid[industry_valid.isin(small_industries)] = "其他"
   ```

3. **共线性处理**：
   ```python
   # 移除常量行业列（避免完全共线）
   constant_cols = [col for col in industry_dummies.columns 
                    if industry_dummies[col].std() < 1e-10]
   industry_dummies = industry_dummies.drop(columns=constant_cols)

   # 变量数 >= 样本数时，移除部分行业dummy
   if X.shape[1] >= X.shape[0]:
       keep_cols = ["const", "log_market_cap"] + list(industry_dummies.columns[:X.shape[0]-3])
       X = X[keep_cols]
   ```

| 问题 | 原因 | 解决办法 |
|:---|:---|:---|
| 小行业 | 某行业股票<2只，无法估计行业beta | 合并到"其他" |
| 常量列 | 某行业全是1，和常数项完全共线 | 删除该行业列 |
| 变量太多 | 行业数 >= 股票数，方程不够 | 只保留大行业 |

**都是为了保证回归可以回归**。

### 3.5 缺失值填补策略

**处理顺序**：
```
1. 因子缺失 → 用行业中位数填补
2. 行业缺失 → 剔除该股票
3. 市值缺失 → 剔除该股票
```

**为何缺行业/市值要剔除**：
- 无法做中性化（需要行业和市值作为回归自变量）
- 这些股票通常是ST/退市/新股，质量较差
