# 02.1 设计原理与逻辑架构
> 因子层（Alpha Factory）- 特征工程核心，负责因子计算与数据清洗

---
## 一、层级职责
### 1.1 核心任务
| 任务 | 说明 | 输出位置 |
|:---|:---|:---|
| 因子计算 | 从原始行情/财务数据计算技术/财务因子 | processed_data/factors/ |
| 数据清洗 | 去极值→填补缺失→中性化→标准化 | 同上（覆盖原文件） |
| PIT对齐 | 财务数据按公告日对齐到交易日历 | processed_data/financial_data/ |

### 1.2 职责边界
- 本层只处理因子计算和清洗，不做模型训练
- 本层保证无未来函数，但不保证因子有效性
- 本层输出标准化因子，均值为0，标准差为1

### 2.3 Pipeline 清洗流程
| 步骤 | 方法 | 输入 | 输出 | 关键参数 |
|:---|:---|:---|:---|:---|
| 去极值 | MAD缩尾法 | 原始因子 | 缩尾后因子 | n_mad=3.0 |
| 填补缺失 | 行业中位数 | 含缺失因子 | 完整因子 | 缺市值/行业则剔除 |
| 中性化 | OLS残差法 | 因子+行业+市值 | 残差Alpha | min_industry_stocks=2 |
| 标准化 | Z-Score | 残差因子 | N(0,1)因子 | - |

**注意：顺序不能更改。**
- 若先去极值后填补，极端值影响行业中位数
- 若先中性化后去极值，极端值影响回归结果
- 标准化必须最后一步

#### 行情数据 → 技术因子（完整流程）
```

(01数据层/raw_data/market_data/)
    │ 000001.SZ.parquet (长格式: time/open/high/low/close/volume/...)
    │ 000002.SZ.parquet
    ▼
┌─────────────────────────────────────┐
│  Step 1: market_data_loader.py      │
│  数据对齐转换                        │
│                                     │
│  • 读取所有个股parquet文件           │
│  • 提取指定字段（如close）           │
│  • 时间戳转换（毫秒→datetime）       │
│  • 拼接成宽表（行:日期，列:股票）     │
└─────────────────────────────────────┘
    │
    ▼
基础数据 (processed_data/market_data/)
    ├─ close.parquet    (收盘价宽表)
    ├─ volume.parquet   (成交量宽表)
    ├─ high.parquet     (最高价宽表)
    └─ ...              (其他字段)
    │
    ▼
┌─────────────────────────────────────┐
│  Step 2: alpha_factory/technical/   │
│  因子计算                            │
│                                     │
│  momentum.py - factor_ret20()       │
│  • 输入: close.parquet              │
│  • 计算: ret20 = close[t]/close[t-20] - 1
│  • 输出: 原始ret20因子宽表           │
│                                     │
│  volatility.py - factor_std20()     │
│  • 输入: close.parquet              │
│  • 计算: 20日收益率标准差            │
│  • 输出: 原始std20因子宽表           │
└─────────────────────────────────────┘
    │
    ▼
原始技术因子 (processed_data/factors/technical/)
    ├─ ret20.parquet    (未清洗)
    ├─ std20.parquet    (未清洗)
    └─ ...
    │
    ▼
┌─────────────────────────────────────┐
│  Step 3: processors/pipeline.py     │
│  因子清洗 (4步流程)                  │
│                                     │
│  依赖数据:                           │
│  • industry.parquet (行业分类)       │
│  • close.parquet + cap_stk.parquet   │
│    → 计算 market_cap = close × cap_stk│
│                                     │
│  清洗流程 (每日截面独立处理):         │
│  ┌─────────────────────────────────┐│
│  │ 1. MAD去极值 (3倍MAD)           ││
│  │    极端值用边界值替换            ││
│  │         ↓                       ││
│  │ 2. 填补缺失值                   ││
│  │    缺行业/市值→剔除; 其他→行业中位数││
│  │         ↓                       ││
│  │ 3. OLS中性化                    ││
│  │    因子 ~ 行业dummy + log(市值)  ││
│  │    输出: 残差 (纯净Alpha)        ││
│  │         ↓                       ││
│  │ 4. Z-Score标准化                ││
│  │    输出: 均值≈0, 标准差≈1        ││
│  └─────────────────────────────────┘│
└─────────────────────────────────────┘
    │
    ▼

    ├─ ret20.parquet    (覆盖, 均值≈0, 标准差≈1)
    ├─ std20.parquet    (覆盖, 均值≈0, 标准差≈1)

```

```
原始财务数据
(01数据层/raw_data/financial_data/)
    │ 000001.SZ.parquet (长格式: report_date/m_anntime/cap_stk/net_profit...)

│  │ • 累计值 → 单季度值 → 4季度滚动和 ││

│                   ↓                 │
│  ┌─────────────────────────────────┐│
│  │ 子步骤1b: PIT对齐               ││

│  │ • 前向填充到下一个公告日         ││
│  │ • 对齐到交易日历（来自close）    ││
│  │                                 ││
│  │ 例: 2010Q1财报4/29公告          ││
│  │     4/29之前 → 用2009Q4数据      ││

│  └─────────────────────────────────┘│
└─────────────────────────────────────┘
    │
    ▼
基础数据 (processed_data/financial_data/)
    ├─ cap_stk.parquet          (期末值，无TTM)
    ├─ tot_shrhldr_eqy.parquet  (期末值，净资产)
    ├─ net_profit_ttm.parquet   (TTM，归母净利润)
    ├─ revenue_ttm.parquet      (TTM，营业收入)
    └─ ...
    │
    ▼
┌─────────────────────────────────────┐
│  Step 2: alpha_factory/financial/   │
│  因子计算                            │
│                                     │
│  valuation.py - factor_pe()         │
│  • 输入:                            │
│    - close.parquet (来自market_data)│
│    - cap_stk.parquet                │

│  • 计算步骤:                         │
│    1. 对齐行列（日期×股票交集）       │
│    2. market_cap = close × cap_stk  │
│    3. PE = market_cap / net_profit_ttm│

│                                     │
│  profitability.py - factor_roe()    │

    │
    ▼

    ├─ pe.parquet       (未清洗)
    ├─ roe.parquet      (未清洗)
    └─ ...
    │
    ▼
┌─────────────────────────────────────┐
│  Step 3: processors/pipeline.py     │
│  因子清洗 (与技术因子相同的4步流程)   │
│                                     │
│  依赖数据:                           │
│  • industry.parquet (行业分类)       │
```

#### 技术因子 vs 财务因子 对比
| 对比项 | 技术因子 | 财务因子 |
|:---|:---|:---|
| **原始数据** | 行情数据（日频） | 财务数据（季度） |
| **预处理** | 宽表转换 | TTM计算 + PIT对齐 |
| **基础数据** | close/volume/... | cap_stk/roe_ttm/... |
| **计算逻辑** | 时序运算（shift/rolling） | 财务公式（乘除法） |
| **需要行情** | 只需要自身 | 需要close计算市值 |
| **清洗流程** | 4步pipeline | 4步pipeline（相同） |
| **关键差异** | 无时序对齐问题 | 必须PIT对齐防未来函数 |

### 2.5 关键数据转换节点
| 节点 | 输入 | 输出 | 关键转换 | 代码位置 |
|:---|:---|:---|:---|:---|
| 行情宽表化 | 个股长格式 | 宽表(日期×股票) | 按时间对齐拼接 | market_data_loader.py |
| PIT对齐 | 季度财报 | 日频宽表 | 公告日填充 | pit_aligner.py |

| 因子清洗 | 原始因子 | 标准化因子 | 4步pipeline | processors/pipeline.py |

## 三、输入输出契约
### 3.1 输入规范（来自01数据层）
| 文件 | 路径 | 格式 | 用途 |
|:---|:---|:---|:---|
| 收盘价 | processed_data/market_data/close.parquet | 宽表(日期×股票) | 动量/估值计算 |
| 成交量 | processed_data/market_data/volume.parquet | 宽表 | 流动性因子 |
| 行业分类 | processed_data/financial_data/industry.parquet | 宽表 | 中性化 |
| 总股本 | processed_data/financial_data/cap_stk.parquet | 宽表 | 市值计算 |
| ROE_TTM | processed_data/financial_data/roe_ttm.parquet | 宽表 | 财务因子 |

### 3.2 输出规范（One Factor One File）
| 属性 | 规范 |
|:---|:---|
| 存储格式 | Parquet |
| 数据结构 | 宽表：行=日期，列=股票代码 |
| 索引 | RangeIndex (0, 1, 2, ...) |
| 首列 | time (datetime64[ns]) |
| 值类型 | float64，缺失值=np.nan |
| 文件命名 | {factor_name}.parquet |
| 存储路径 | processed_data/factors/{technical/financial}/ |

### 3.3 清洗后因子标准
| 指标 | 标准值 | 检查方法 |
|:---|:---|:---|
| 均值 | ≈0 | factor.mean() |
| 标准差 | ≈1 | factor.std() |
| 缺失值比例 | <50% | factor.isna().sum() |
| 与行业相关性 | ≈0 | 分组均值检验 |
| 与市值相关性 | ≈0 | pearson相关系数 |

## 五、核心设计决策
### 5.1 为什么One Factor One File？
| 方案 | 优点 | 缺点 | 本项目选择 |
|:---|:---|:---|:---|
| 多因子单文件 | 文件少，加载快 | 新增/修改因子需重写整个文件 | ❌ |
| One Factor One File | 灵活，按需加载 | 文件多 | ✅ |

理由：
- 模型训练时通常只需部分因子，按需加载节省内存
- 新增因子无需修改已有文件，避免数据回刷风险
- 便于版本控制和增量更新

### 5.2 为什么MAD去极值？
| 方法 | 优点 | 缺点 | 本项目选择 |
|:---|:---|:---|:---|
| 3σ法则 | 简单 | 对极端值敏感，不稳健 | ❌ |
| MAD缩尾 | 稳健，不受极端值影响 | 计算稍复杂 | ✅ |
| 分位数缩尾 | 简单直观 | 损失分布信息 | ❌ |

MAD公式：
```
median = factor.median()
mad = (factor - median).abs().median()
lower = median - 3 * 1.4826 * mad
upper = median + 3 * 1.4826 * mad
```

### 5.3 为什么OLS中性化？
模型：`因子 = β₀ + Σβᵢ×行业dummyᵢ + βₘ×log(市值) + ε`

输出残差ε作为纯净Alpha，剥离：
- 行业Beta（不同行业估值水平差异）
- 市值Beta（大小盘风格差异）

### 5.4 为什么每日截面独立处理？
- 防止跨天信息泄露（如用未来数据标准化）
- 保持时序独立性，符合金融数据特性
- 便于并行化加速

## 六、关键概念解释
### 6.1 PIT（Point-in-Time）对齐
**问题**：财务数据是季度报告，如何对齐到日频？

**方案**：按公告日(m_anntime)前向填充
```
财报1(2010Q1, 4/29公告) ─────── 财报2(2010Q2, 8/25公告)
     │                              │
     ▼                              ▼
2010-04-29 ~ 2010-08-24       2010-08-25 ~ 2010-10-27
   使用财报1数据                 使用财报2数据
```

**关键**：财报2公告后，市场才知晓Q2数据，因此8/25之后才更新

### 6.2 TTM（Trailing Twelve Months）计算
**流量表指标**（利润表）需要计算TTM：

```
步骤1: 累计值 → 单季度值
   Q1累计 = Q1
   Q2累计 = Q1+Q2 → Q2单季 = Q2累计 - Q1累计
   Q3累计 = Q1+Q2+Q3 → Q3单季 = Q3累计 - Q2累计
   Q4累计 = 全年 → Q4单季 = Q4累计 - Q3累计

步骤2: 单季度 → TTM
   TTM(t) = 单季(t) + 单季(t-1) + 单季(t-2) + 单季(t-3)
```

**时点表指标**（资产负债表）直接用期末值，不需要TTM。

### 6.3 未来函数检查点
| 检查项 | 方法 | 通过标准 |
|:---|:---|:---|
| 真未来函数 | 最后一天值是否为空 | 应为空（用到t+1价格） |
| 统计泄露 | 标准化/去极值是否跨天 | 每日截面独立处理 |
| 财务数据泄露 | PIT对齐是否正确 | 公告日之后才能使用 |
