## 一、系统概览

### 1.1 系统定位

构建一个**可复现、可维护、可快速迭代**的截面多因子量化选股系统，通过AI模型（LightGBM/XGBoost）捕捉因子与收益率之间的非线性关系，实现超越基准指数的超额收益（Alpha）。

### 1.2 核心准则

> **"逻辑领先于代码，相关性服务于因果。"**

- 每个因子必须有清晰的因果逻辑（Causal Logic）
- 通过15-20个低相关因子流实现Alpha与Beta的分离
- 追求"交易圣杯"：跨周期的Alpha捕捉机器

## 二、架构原则

### 2.1 存算分离

```
数据存储层 (Storage)          计算层 (Compute)
┌─────────────────┐           ┌─────────────────┐
│   Parquet文件   │           │   Python进程    │
│   (磁盘/对象存储)│◄─────────→│   (内存计算)    │
└─────────────────┘           └─────────────────┘
```

- 数据以Parquet格式存储，计算时加载到内存
- 支持增量更新，避免重复计算
- 便于版本控制和回溯

### 2.2 低耦合设计

| 层级 | 职责 | 依赖关系 |
|------|------|----------|
| 数据层 | 原始数据存储 | 无依赖 |
| 因子层 | 因子计算与清洗 | 依赖数据层输出 |
| 模型层 | 特征组合与预测 | 依赖因子层输出 |
| 回测层 | 信号执行与绩效 | 依赖模型层输出 |

### 2.3 One Factor, One File

每个因子存储为独立的Parquet宽表：

```
processed_data/factors/
├── ret20.parquet          # 20日收益率因子
├── ret60.parquet          # 60日收益率因子
├── pe_ttm.parquet         # 市盈率TTM
├── roe_ttm.parquet        # ROE TTM
└── ...
```

**文件格式规范**:
- **Index**: 时间 (DateTime64[ns])
- **Columns**: 股票代码 (Ticker)
- **Values**: 该股票在当天的因子值

**代码实现** (`processors/pipeline.py`):

```python
def clean_factor(factor, industry, market_cap, steps=None):
    """
    一键清洗因子 (单截面)
    流程: 去极值 → 填补缺失 → 中性化 → 标准化
    """
    if steps is None:
        steps = ['outlier', 'missing', 'neutralize', 'standardize']
    
    result = factor.copy()
    
    if 'outlier' in steps:
        result = mad_winsorize(result, n_mad=3.0)
    
    if 'missing' in steps:
        result, industry_clean, market_cap_clean = fill_missing(result, industry, market_cap)
    
    if 'neutralize' in steps:
        result = neutralize(result, industry_clean, market_cap_clean)
    
    if 'standardize' in steps:
        result = zscore_standardize(result)
    
    return result
```

#### 3.4.3 信号生成逻辑

```python
def generate_signals(df, rebalance_period=20, stocks_per_batch=30):
    """
    生成买卖信号
    
    参数:
    - rebalance_period: 调仓周期(交易日)
    - stocks_per_batch: 每期选股数量
    
    返回:
    - buy_date: {日期: [股票列表]}
    - sell_date: {日期: [股票列表]}
    """
    rebalance_dates = get_trading_days()[::rebalance_period]
    
    for date in rebalance_dates:
        # 获取当天截面数据
        current_slice = df.loc[date]
        
        # 按预测分数排序，选前N只
        selected = current_slice.sort_values('prediction', ascending=False)
        buy_list = selected.head(stocks_per_batch)['stock_code'].tolist()
        
        # 卖出不在新列表中的持仓
        sell_list = list(current_position - set(buy_list))
        
        buy_date[date] = buy_list
        sell_date[date] = sell_list
    
    return buy_date, sell_date
```

### 6.1 模型基类接口

```python
from abc import ABC, abstractmethod
from typing import Optional
import pandas as pd
import numpy as np

class BaseModel(ABC):
    """模型基类 - 定义统一接口"""
    
    def __init__(self, config: dict):
        self.config = config
        self.is_fitted = False
    
    @abstractmethod
    def fit(
        self, 
        X_train: pd.DataFrame, 
        y_train: pd.Series,
        X_valid: Optional[pd.DataFrame] = None,
        y_valid: Optional[pd.Series] = None
    ) -> 'BaseModel':
        """训练模型"""
        pass
    
    @abstractmethod
    def predict(self, X_test: pd.DataFrame) -> np.ndarray:
        """预测得分"""
        pass
    
    @abstractmethod
    def get_feature_importance(self) -> pd.Series:
        """获取特征重要性"""
        pass
    
    def save(self, path: str):
        """保存模型"""
        import pickle
        with open(path, 'wb') as f:
            pickle.dump(self, f)
    
    @classmethod
    def load(cls, path: str):
        """加载模型"""
        import pickle
        with open(path, 'rb') as f:
            return pickle.load(f)
```

## 附录

### A. 术语表

| 术语 | 英文 | 解释 |
|------|------|------|
| PIT对齐 | Point-in-Time | 按公告日将财务数据对齐到交易日历，避免未来函数 |
| TTM | Trailing Twelve Months | 滚动12个月，用于将累计财务数据转为年度可比数据 |
| 等比前复权 | Front Ratio Adjustment | 使用乘法公式调整历史价格，不会出现负数 |
| 中性化 | Neutralization | 通过回归剥离行业和市值Beta，保留纯Alpha |
| MAD | Median Absolute Deviation | 中位数绝对偏差，用于去极值 |
| Walk-forward | 滚动训练 | 用历史数据训练，预测未来数据，防止过拟合 |
| IC | Information Coefficient | 信息系数，因子与收益率的相关系数 |
| IR | Information Ratio | 信息比率，IC的均值/标准差 |
