# 01.3 数据层 - 运维与变更日志
> 文档定位：运维检查、问题排查、记录变更和待办事项
> 更新频率：高（持续记录）

---
## 1. 关键检查点
### 1.1 数据完整性检查
| 检查项 | 检查方法 | 通过标准 | 异常处理 |
|:---|:---|:---|:---|
| 行情数据日期连续 | 检查每个股票的时间序列是否连续 | 无缺失交易日 | 补录缺失数据 |
| 财务数据PIT对齐 | 检查 m_anntime 是否晚于 report_date | 无未来数据 | 重新对齐 |
| 股票池一致性 | 对比股票列表与基准指数成分股 | 与基准一致 | 更新股票池 |

### 1.2 数据质量检查
| 检查项 | 检查方法 | 通过标准 | 异常处理 |
|:---|:---|:---|:---|
| 价格范围合理 | 检查 close 是否在合理范围内 | 0 < price < 1e4 | 标记异常并核查 |
| 成交量非负 | 检查 volume >= 0 | volume ≥ 0 | 修正或删除 |
| 无重复记录 | 检查 (code, date) 唯一性 | 唯一索引不冲突 | 去重处理 |

## 2. 性能基准
| 指标 | 基准值 | 测试环境 | 备注 |
|:---|:---|:---|:---|
| 全量下载耗时 | 2-4小时 | 全市场5000+股票 | 取决于网络状况 |
| 单日增量耗时 | 5-10分钟 | 新增1个交易日 | 仅下载新数据 |
| 内存占用 | < 2GB | 单进程运行 | 可通过分批控制 |
| 存储空间 | ~10GB/年 | 行情+财务+元数据 | 可通过压缩优化 |

## 3. 运维操作步骤
### 3.1 首次初始化
```bash
cd 01数据
python data_main.py --full
```

检查点：
- [ ] 所有股票行情数据已下载
- [ ] 财务数据包含历史所有报告期
- [ ] 行业映射文件已生成
- [ ] update_log.json 已创建

### 3.2 每日增量更新
```bash
cd 01数据
python data_main.py --monthly
```

检查点：
- [ ] 当日行情数据已更新
- [ ] 无新增财务报告（非财报季）
- [ ] update_log.json 已更新

### 3.3 月度全量更新
```bash
cd 01数据
python data_main.py --full
```

检查点：
- [ ] 行情数据全量重新下载完成
- [ ] 财务数据已合并更新
- [ ] 元数据已刷新

### 3.4 故障回滚
| 场景 | 回滚步骤 | 预计耗时 |
|:---|:---|:---|
| 数据损坏 | 从备份恢复或使用 --full 重新下载 | 2-4小时 |
| 部分股票缺失 | 针对性补录缺失股票 | 10-30分钟 |
| 财务数据错误 | 删除错误文件，重新下载 | 30-60分钟 |

## 附录：常见问题排查
### 问题 1：下载速度过慢
现象：下载全市场数据超过6小时仍未完成

排查步骤：
1. 检查网络连接是否稳定
2. 检查 BATCH_SIZE 是否设置过大
3. 检查是否选择了最优服务器

解决：
- 减小 BATCH_SIZE 到 200
- 增大 BATCH_SLEEP 到 2秒
- 更换更近的服务器
