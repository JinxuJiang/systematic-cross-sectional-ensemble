# 00.1 项目总体要求
> 截面多因子量化选股系统 - 项目愿景、架构设计与规范标准

---
## 一、项目愿景
### 1.1 核心目标
构建一个可复现、可维护、可快速迭代的截面多因子量化选股系统。

通过引入机器学习、深度学习的模型（LightGBM/XGBoost/Transformer）捕捉因子与收益率之间的非线性关系，实现超越基准指数的超额收益。

可实现扩展的选股系统。后续能够引入高级因子（自动挖掘的因子）、高级模型（训练更好的模型）。

### 1.2 核心准则
> 逻辑领先于代码，相关性服务于因果。

- 每个因子必须有清晰的因果逻辑（Causal Logic）
- 通过低相关因子流实现 Alpha 与 Beta 的分离

---
## 二、系统架构
整个系统分为四个核心模块，遵循"存算分离"和"低耦合"原则：

```
                     截面多因子量化选股系统                        

  Module 1        Module 2         Module 3        Module 4      
                    
    Data     Alpha    Alpha  Optimizer     
   Engine       Factory      Combiner     &Backtest     
   数据引擎      因子工厂       AI训练      优化回测      
                                                              
   原始数据        因子计算         非线性建模      组合优化       
   PIT对齐        数据清洗         滚动训练        交易成本       
```

### 2.1 模块职责

| 模块 | 英文名 | 核心职责 | 关键技术 |
|:---|:---|:---|:---|
| 模块1 | Data Engine | 原始数据搬运与存储 | Parquet格式、原始数据的PIT对齐（通过ann_time和rep_time处理财务数据修正） |
| 模块2 | Alpha Factory | 因子计算与数据清洗 | PIT对齐、去极值、标准化、中性化 |
| 模块3 | Alpha Combiner | AI模型训练与集成 | LightGBM/XGBoost、Walk-forward |
| 模块4 | Optimizer & Backtest | 组合优化与回测 | 行业中性、换手率约束 |

### 2.2 模块详解

#### Module 1: Data Engine (数据引擎)
职责：原始行情（OHLCV）与财务数据（PIT格式）的搬运与存储

关键设计：
- 存储格式：本地 Parquet（按股票代码索引），兼顾读取速度与压缩率
- 复权方式：等比前复权（front_ratio），避免传统前复权的负数问题
- 财务数据：采用原始数据的PIT对齐，通过ann_time和rep_time处理财务数据中某一期对之前数据修改等问题

#### Module 2: Alpha Factory (因子工厂)
职责：特征工程核心，负责因子计算、数据清洗、中性化处理

关键设计：
- 因子分类：按家族分类（技术因子、财务因子、机器学习因子、论文因子），后续扩展方便
- 因果逻辑：每个因子必须包含经济学因果解释
- PIT对齐：确保财务数据在计算时已公告，并对齐到行情数据的日频。行情数据和财务数据的PIT对齐做在这里，方便特征工程

#### Module 3: Alpha Combiner (AI 训练与集成)
职责：使用机器学习、深度学习的模型对多维度因子进行权重分配与非线性建模

关键设计：
- 模型选择：LightGBM（处理非线性和缺失值）、XGBoost、Transformer
- 训练策略：Walk-forward（滚动训练），防止过拟合
- 验证方式：截面回归，预测下个月收益

#### Module 4: Optimizer & Backtest (组合优化与回测)
职责：将模型输出的预测分数转化为交易头寸

关键设计：
- 组合优化：考虑行业中性、换手率约束
- 策略回测：
  - alphalens回测：检验因子有效性
  - backtrader回测：模拟实盘交易，设置交易成本、滑点等
- 风险指标：IC表现、最大回撤、夏普比率、胜率等

### 3.2 数据存储规范

标准：One Factor, One File

所有的数据都存在 processed_data，包括基础数据、因子数据。每个因子存储为独立的 Parquet 宽表：

```
processed_data/
├── market_data/              # 行情基础数据 (open/high/low/close...)
├── financial_data/           # 财务基础数据 (PIT对齐后的原始字段)
└── factors/                  # 因子数据
    ├── technical/            # 技术因子 (ret20/rsi/macd...)
    └── financial/            # 财务因子 (pe/pb/roe...)
```

文件格式规范：
- Index: 索引（0-N整数）
- Columns: 时间（date）、股票代码（ticker）及因子值
  - 时间保存为单独一列
  - 股票代码作为列名
- Values: 对应股票在当天的因子值

## 四、各模块架构设计
### 4.1 Module 2: Alpha Factory (因子工厂) 架构

设计原则：
- "一家族一文件"：按因子家族（技术/财务/质量）聚合
- 因果逻辑优先：每个因子必须包含经济学因果解释 Docstring

目录结构：
```
alpha_factory/
├── technical/                # 技术因子家族
│   ├── momentum.py           # 动量因子 (ret20/ret60...)
│   ├── volatility.py         # 波动率因子
│   └── liquidity.py          # 流动性因子
└── financial/                # 财务因子家族
    ├── valuation.py          # 估值因子 (PE/PB/PS)
    ├── profitability.py      # 盈利因子 (ROE/ROA)
    ├── quality.py            # 质量因子 (周转率/现金流)
    └── growth.py             # 成长因子 (YoY增长)
```

因子计算流程：
```
原始数据（01数据层/raw_data）
    ↓
PIT数据 (processed_data/market_data, processed_data/financial_data)
    ↓
┌─────────────────┐
│  因子家族类      │  ← 加载所需数据
│  (XXXFactors)   │
└────────┬────────┘
         ↓
┌─────────────────┐
│  因子计算方法    │  ← 因果逻辑计算
│  factor_xxx()   │
└────────┬────────┘
         ↓
┌─────────────────┐
│  Pipeline清洗   │  ← 去极值/标准化/中性化
└────────┬────────┘
         ↓
┌─────────────────┐
│  输出因子宽表    │  → 保存到 processed_data/factors/
│  (因子值矩阵)    │
└─────────────────┘
```

### 4.2 Module 3: Alpha Combiner 架构

设计目标：

```
时间轴:  2010    2015    2020    2024

             2011-2015    2016

               2012-2016    2017
                ...
```

### 4.3 Module 4: Optimizer & Backtest (回测) 架构

设计目标：
- 组合优化：将预测分数转化为实际可交易头寸
- 风险控制：行业中性、换手率约束
- 策略表现：alphalens检查因子能力，backtrader回测看策略表现

核心组件：
```
04回测层/
├── alphalens_analysis.py       # Alphalens因子分析
├── backtrader_eval_*.py        # Backtrader策略回测
├── generate_live_signals.py    # 生成实盘信号
└── utils.py                    # 工具函数
```

回测流程：
```
输入: 模型预测的因子分数 (每日截面)
    ↓
┌─────────────────┐
│  组合优化        │  ← 考虑约束条件生成目标权重
│  (Optimizer)    │
└────────┬────────┘
         ↓
┌─────────────────┐
│   Alphalens分析  │  ← 检验因子有效性 (IC/IR)
│  (Factor Analysis)
└────────┬────────┘
         ↓
┌─────────────────┐
│  Backtrader回测  │  ← 模拟实盘交易
│  (Backtest)     │      考虑滑点、手续费
└────────┬────────┘
         ↓
┌─────────────────┐
│   绩效分析       │  ← 夏普比率、最大回撤等
│  (Metrics)      │
└─────────────────┘
```

## 五、数据标准
### 5.1 行情数据标准

| 属性 | 规范 |
|:---|:---|
| 频率 | 日频 (1d) |
| 复权 | 等比前复权 (front_ratio) |
| 列名 | time, open, high, low, close, volume, amount |
| 时间戳 | datetime.date (YYYY-MM-DD) |
| 存储 | 宽表格式（行：日期，列：股票） |

### 5.2 财务数据标准

| 属性 | 规范 |
|:---|:---|
| 格式 | PIT (Point-in-Time) |
| 对齐键 | m_anntime (公告日), report_date (报告期) |
| 处理 | TTM计算（流量表）、期末值（时点表） |
| 存储 | 宽表格式（PIT对齐到日频） |

### 5.3 元数据标准

| 属性 | 规范 |
|:---|:---|
| 行业分类 | 申万一级行业（31个行业） |
| 股票池 | 全A股 / 沪深300 / 中证500 / 中证1000 |

### 5.4 数据检验标准

分层检验：
1. 每层逻辑检验，数据生成逻辑是否正确
2. 数据泄露检验
   - 若最后一日值为空：真未来函数（用到t+1价格/收益）
   - 最后一日值不为空：全样本统计泄露（标准化/去极值用了未来信息）

检验类型：
- 完整性检验（缺失值比例）
- 一致性检验（日期范围、股票代码对齐）
- 业务规则检验（负数检查、异常值检测）

报告格式：Markdown 报告，包含通过/失败状态
